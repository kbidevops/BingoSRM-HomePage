취약점 유형 소개

CCE 취약점과 CVE 취약점

CCE = "설정 잘못" | CVE = "소프트웨어 제품 버그"
CCE
CCE (Common Configuration Enumeration) 개요
CCE는 시스템과 소프트웨어의 보안 설정 오류(Configuration Weakness)를 표준 식별번호로 관리하는 체계입니다.
KISA의 취약점 표준 체계
대한민국에서는 한국인터넷진흥원(KISA)이 주요 정보보호 취약점 기준을 관리·보급합니다. 다음과 같은 국내 표준 취약점 점검 기준을 제공합니다.
① 주요정보통신기반시설 기술적 취약점 분석·평가 기준
  • 전자정부, 공공기관 대상
  • 서버/네트워크/DB/보안장비 점검 항목 제공
  • 항목 구조는 CCE 개념과 유사 (설정 기반 점검)
② 국가 정보보안 기본지침 / 전자정부 보안 가이드
  • 계정관리
  • 접근통제
  • 로그관리
  • 암호정책
CVE
CVE (Common Vulnerabilities and Exposures) 개요
소프트웨어나 하드웨어 제품에 존재하는 보안 취약점(버그)에 부여되는 고유 식별번호입니다.
CVE 취약점의 주요 내용
  • CVE 의미: 소프트웨어 결함
  • CVE 대상: 소프트웨어 제품 코드 취약점
  • CVE 약점: 해커가 직접 공격 가능
  • CVE 해결: 소프트웨어 취약점 점검 및 패치
CVE의 특징
  • 실제 공격 가능한 코드 취약점
  • 형식: CVE-연도-번호
    예: CVE-2024-12345
  • CVSS 점수와 연계 (위험도 평가)
CVE 예시
CVE-2021-44228
→ Apache Log4j의 Log4Shell 원격 코드 실행 취약점
CVE-2017-0144
→ Microsoft Windows SMB 취약점 (WannaCry 악용)

BingoCVM 시스템 구성
